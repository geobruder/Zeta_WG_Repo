## Per‑state filled counts (group imputation audit)

```{r per-state-filled, echo=TRUE, fig.height=4}
if (!is.null(group_audit)) {
  filled_counts <- enframe(group_audit$filled_counts, name = "feature", value = "filled_count")
  filled_counts %>% arrange(desc(filled_count)) %>% kable(caption = "Counts of values filled by group imputation (per feature)") %>% kable_styling(full_width = FALSE)
  if (!is.null(group_audit$group_col_used) && group_audit$group_col_used == "state" && file_exists(data_clean)) {
    df <- read_csv(data_clean, show_col_types = FALSE)
    num_cols <- df %>% select(where(is.numeric)) %>% names()
    top3 <- head(num_cols, 3)
    if (length(top3) > 0) {
      df %>% mutate(state = if_else(!is.na(state), state, substr(fips,1,2))) %>%
        group_by(state) %>%
        summarise(across(all_of(top3), ~ sum(is.na(.x)))) %>%
        pivot_longer(-state, names_to="feature", values_to="n_missing") %>%
        ggplot(aes(x=reorder(state, -n_missing), y=n_missing, fill=feature)) +
        geom_col(position="dodge") +
        labs(x="State", y="Missing count filled", caption="Per-state missing counts for representative features (filled by group imputation)") +
        theme_minimal(base_size=11) +
        theme(axis.text.x = element_text(angle=90, vjust=0.5, hjust=1))
    }
  } else {
    cat("State column not present in analysis_table.csv or group imputation used global fallback; showing global fills only.\n")
  }
} else {
  cat("group_imputation_audit.json not found; run group imputation cell in notebook.\n")
}
```

**Figure 1. Per‑state missing counts filled by group imputation.**  
*Description.* Bar chart showing the number of missing values filled per state for the top numeric features (features plotted in separate colors). States are ordered by descending filled count. *Interpretation.* The plot shows which states contributed most to the imputation counts for the selected numeric features; a small number of states account for the majority of filled values, indicating geographic clustering of missingness. This suggests that group (state) median imputation leverages meaningful geographic structure — states with high filled counts should be inspected for systematic data collection gaps and considered when interpreting model predictions for those states.

\newpage

## Imputation strategy and leakage check

**What we did**

- Numeric features: median imputation (robust to skew and outliers).  
- Missingness indicators: added for each numeric feature to preserve missingness signal.  
- Group (state) median imputation: where a state median exists, use it; otherwise fallback to global median.

## Unsupervised feature engineering example — PCA

```{r pca-example, echo=TRUE, fig.height=4.5}
if (file_exists(data_clean)) {
  df <- read_csv(data_clean, show_col_types = FALSE)
  num_df <- df %>% select(where(is.numeric)) %>% select(-coverage_estimate)
  if (ncol(num_df) >= 2) {
    num_scaled <- scale(na.omit(num_df))
    pca <- prcomp(num_scaled, center = TRUE, scale. = TRUE)
    var_explained <- pca$sdev^2 / sum(pca$sdev^2)
    qplot(seq_along(var_explained), var_explained, geom="line") +
      geom_point() +
      labs(x="Principal component", y="Proportion variance explained", caption="PCA scree plot (numeric features)") +
      theme_minimal(base_size=11)
    loadings <- as.data.frame(pca$rotation)
    write_csv(loadings, path(outputs_dir, "pca_loadings.csv"))
  } else {
    cat("Not enough numeric features for PCA.\n")
  }
} else {
  cat("analysis_table.csv not found; cannot run PCA.\n")
}
```

**Figure 2. PCA scree plot (numeric features).**  
*Description.* Line plot of proportion of variance explained by each principal component. *Interpretation.* The scree plot (if produced) indicates how variance is distributed across components; a steep drop after the first few components would indicate that a small number of components capture most variance. In the current run the notebook reported "Not enough numeric features for PCA", so no PCA loadings were produced — this indicates the dataset has few numeric predictors after preprocessing or many are logical/flag variables; supervised feature engineering (model‑based selection) is therefore more appropriate here.

## Discussion and next steps

- The preprocessing pipeline is conservative and auditable. Group imputation leverages geographic structure; the sensitivity check (imputer inside pipeline) is included to detect leakage risk.  
- Next steps: (1) run supervised feature selection (e.g., recursive feature elimination or model‑based selection) on training folds; (2) produce per‑county SHAP reports for top counties; (3) consider temporal holdout if time series data is available.
